# 总结与思考
回答导言中的问题：
首先放弃Word，改用Markdown，因为文本文件容易比较、查看冲突。每个人建一个自己的分支，完成自己的部分并推到远程仓库，审核者再建一个dev分支，审核后一一把最新的个人分支合并到dev里面，其他人拉取dev分支并合并到自己的分支，重复这个过程（提交时注明自己的看法）。待所有人无异议后审核者把dev合并到master/main里面。
